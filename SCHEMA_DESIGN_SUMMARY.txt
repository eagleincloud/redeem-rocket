PERFORMANCE OPTIMIZATION
================================================================================

INDEXING STRATEGY:
  • All foreign keys indexed for JOIN performance
  • All filter columns indexed with predicates
  • Timestamps indexed DESC for recent-first queries
  • Composite indexes for common patterns
  • Partial indexes for active-only records
  • JSONB indexes for common paths (optional)

PARTITIONING STRATEGY:
  • behavior_tracking: Monthly partitions (keeps working set small)
  • feature_usage_stats: Monthly partitions (easy archival)
  • Enables fast queries on recent data
  • Facilitates archival of old data to cold storage

MATERIALIZED VIEWS:
  • v_category_adoption_metrics (refresh nightly)
  • v_template_performance (refresh nightly)
  • Pre-computed joins reduce query complexity

QUERY OPTIMIZATION:
  • RLS via indexed EXISTS subqueries (not full table scan)
  • Views for common join patterns
  • Pagination for large result sets
  • Batch operations supported

================================================================================
SCALABILITY PATH
================================================================================

FOR 10k BUSINESSES:
  • Use schema as-is
  • Single database instance sufficient
  • Monitor query latencies
  • Daily stats aggregation completes < 5 minutes

FOR 100k BUSINESSES:
  • Implement monthly partitioning on behavior_tracking
  • Implement monthly partitioning on feature_usage_stats
  • Add read replicas for analytics queries
  • Use PgBouncer for connection pooling (25 default, 500 max)
  • Archive behavior data > 1 year old to cold storage

FOR 1M+ BUSINESSES:
  • Horizontal sharding by category or region
  • Separate analytics database (read-only replica)
  • Event streaming (Kafka) for behavior tracking
  • Elasticsearch for complex search
  • Data warehouse (BigQuery) for long-term trends
